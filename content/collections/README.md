# Collections (internal taxonomy)

`data/collections.ts` groups objects for **editors and future CMS** — e.g. matching ritual-adjacent sets or merchandising rules.

The **public Quiet Ash hierarchy** is intentionally only:

1. **Rituals** — lived atmosphere + rhythm + objects in context  
2. **Objects** — commerce  
3. **Essays** — philosophy + SEO  

Routes under `/collections` may remain for legacy links or admin previews; they are **not** a primary navigation pillar.
